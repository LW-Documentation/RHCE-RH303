Bi 6. DCH V TN MIN

1.           Gii thiu

-         My tnh v cc thit b kt ni mng, c th l cc giao din mng (network interfaces) ca chng, s dng a ch IP lm nh danh. Con ngi s dng tn min thay cho IP d nh. V vy, cn phn gii/nh x t tn min sang a ch IP v ngc li.

-         DNS (Domain Name System) l mt h c s d liu phn tn dng nh x gia cc tn min v cc a ch IP.

-         DNS Client: Chy trn thit b ca ngi dng, nhn tn min do ngi dng cung cp, v d ngi dng nhp a ch trang web trn trnh duyt, hi DNS bit a ch IP ca my ch.

-         DNS Server: H thng phn cp cc DNS servers.

2.2. Cc phng php c bn kim tra DNS

DNS phn gii tn min y (FQDN - Fully Qualified Domain Name), v d nh vnu.edu.vn, ti mt a ch IP. Vic ny c gi l phn gii xui. Phn gii t a ch IP ra tn min y c gi l phn gii ngc.

Nhiu tn min c th nh x ti mt a ch IP duy nht, nhng ngc li th khng ng, mt a ch IP ch c th nh x ti duy nht mt tn min. iu ny c ngha vic phn gii xui v ngc c th khng cho kt qu ging nhau. Cc trng trong DNS dng cho phn gii ngc thng thuc trch nhim ca nh cung cp dch v (ISP), nh vy l kh ph bin cho vic phn gii ngc ti tn min ca ISP. y khng phi l yu t quan trng i vi cc trang web nh, nhng vi mt s ng dng thng mi in t thng yu cu vic phn gii xui v ngc phi ph hp vi nhau. Ta c th yu cu ISP thay i DNS cho ph hp vi yu cu ny.

Di y l mt s lnh ta c th dng tra cu tn min.

2.2.1. Lnh host

Lnh host chp nhn cc tham s l tn min y hoc a ch IP. tm phn gii xui (tm a ch IP ca tn min y ), ta s dng c php nh sau:

[root@localhost ~]# host www.linuxhomenetworking.com

www.linuxhomenetworking.com has address 65.115.71.34

Cn tm phn gii ngc (tm tn min y t a ch IP)

[root@localhost ~]# host 65.115.71.34

34.71.115.65.in-addr.arpa domain name pointer 65-115-71-34.myisp.net.

2.2.2. Lnh nslookup

Lnh nslookup dng tm kim a ch IP t tn min

[root@localhost ~]# nslookup www.linuxhomenetworking.com

Non-authoritative answer:

Name:    www.linuxhomenetworking.com

Address:  65.115.71.34

 

thc hin tm kim tn min t a ch IP:

3.             DNS Server

DNS Server c th cung cp cc thng tin do DNS client yu cu, hoc chuyn n mt DNS Server khc nh phn gii h trong trng hp n khng th tr li c cc truy vn v nhng tn min khng thuc quyn qun l, ng thi cng lun sn sng tr li cc DNS server khc v cc tn min m n qun l. DNS server cp cao nht l Root Server do t chc ICANN qun l.

o   Phn loi:

  Primary DNS server

  Secondary DNS server

  Caching DNS  server

Khuyn ngh nn s dng t nht l hai DNS Server lu cho mi mt zone. Primary DNS Server qun l cc zone v Secondary Server lu tr d phng cho Primary Server. Secondary DNS Server c khuyn ngh dng nhng khng nht thit phi c.

Caching  DNS  server ch s dng cho vic truy vn, lu gi cu tr li da trn thng tin c trn cache ca my v cho kt qu truy vn. Chng khng h qun l mt domain no v thng tin m n ch gii hn nhng g c lu trn cache ca Server.

3.1. Ci t DNS Server BIND

BIND (Berkeley Internet Name Distributed) l phn mm DNS Server c s dng nhiu nht hin nay trn th gii. Mt chng trnh phc v DNS trn nn cc h thng AIX/BSD/HP-UX/Unix/Linux..., Bind chy nhanh, hiu sut cao, p ng c s lng user ln, cu hnh linh hot, ...

Chy  lnh  sau ci t  BIND

sudo apt install bind9 dsnutils

3.2. Cu hnh BIND

Th mc cha cc tp cu hnh l /etc/bind/. Tp cu hnh chnh l /etc/bind/named.conf. Cc dng include bao hm cc tp cha cc ty chn.

3.2.1. Cu hnh Caching nameserver

Nu ch s dng bind nh caching nameserver, n gin ch cn thm cc a ch IP ca cc nameservers khc (v d ca cc ISP). Chnh sa tp /etc/bind/named.conf.options, b ch thch (#) v chnh sa ni dung forwarders tng t sau:

forwarders {

            8.8.8.8;

            8.8.4.4;

};

cu hnh mi c tc dng, hy khi ng li bind bng cch chy lnh sau:

            sudo systemctl restart bind9.service

Phn gii xui

bt u cu hnh phn gii xui cho min example.com, thm zone xui mi nh sau vo tp /etc/bind/named.conf.local:

zone "example.com" {

 type master;

file "/etc/bind/db.example.com";

};

By gi, to tp /etc/bind/db.example.com v a cc bn ghi DNS vo tp ny nh v d sau:

$TTL    604800

@       IN      SOA     ns.example.com. root.example.com. (

                              2         ; Serial

                         604800         ; Refresh

                          86400         ; Retry

                        2419200         ; Expire

                         604800 )       ; Negative Cache TTL

        IN      A       192.168.1.10

;

@       IN      NS      ns.example.com.

@       IN      A       192.168.1.10

@       IN      AAAA    ::1

ns       IN      A       192.168.1.10

printer IN A 192.168.1.11

hoapc IN A 192.168.1.12

ftp IN CNAME hoapc.example.com

cu hnh mi c tc dng, hy khi ng li bind bng cch chy lnh sau:

            sudo systemctl restart bind9.service

Phn gii ngc

bt u cu hnh phn gii ngc cho min example.com, thm zone ngc mi nh sau vo tp /etc/bind/named.conf.local:

zone "1.168.192.in-addr.arpa" {

        type master;

        file "/etc/bind/db.192";

};

Thay 1.168.192 bng ba octet u ca mng (vit ngc). ng thi t tn tp ca zone l /etc/bind/db.192 vi s l octet u ca mng.

By gi to tp  /etc/bind/db.192 v a cc bn ghi DNS vo tp ny nh v d sau:

$TTL    604800

@       IN      SOA     ns.example.com. root.example.com. (

                              2         ; Serial

                         604800         ; Refresh

                          86400         ; Retry

                        2419200         ; Expire

                         604800 )       ; Negative Cache TTL

;

@       IN      NS      ns.

10      IN      PTR     ns.example.com.

11      IN      PTR     printer.example.com.

12      IN      PTR     hoapc.example.com.

 

Mi bn ghi A trong /etc/bind/db.example.com cn c mt bn ghi PTR trong /etc/bind/db.192.

cu hnh mi c tc dng, hy khi ng li bind bng cch chy lnh sau:

            sudo systemctl restart bind9.service

 

3.2.3. Cu hnh nameserver th cp

Secondary DNS Server nn c thit lp m bo tnh sn sng ca dch v DNS. thit lp DNS server th cp, trc ht cn cu hnh DNS server chnh n c php chuyn zone. Thm ty chn allow-transfer vo c zone xui v zone ngc nh sau:

zone "example.com" {

        type master;

            file "/etc/bind/db.example.com";

        allow-transfer { 192.168.1.13; };

};

zone "1.168.192.in-addr.arpa" {

        type master;

        file "/etc/bind/db.192";

            allow-transfer { 192.168.1.13; };

};

Trong v d ny, DNS server th cp s c a ch IP l 192.168.1.13.

Khi ng li DNS server chnh cu hnh mi c tc dng.

Tip theo, trn DNS server th cp, ci t BIND nh bnh thng, sau chnh sa tp /etc/bind/named.conf.local v thm c zone xui v zone ngc nh sau:

zone "example.com" {

            type slave;

        file "db.example.com";

        masters { 192.168.1.10; };

};       

     

zone "1.168.192.in-addr.arpa" {

            type slave;

        file "db.192";

        masters { 192.168.1.10; };

};

Lu 192.168.1.10  l a ch IP ca nameserver chnh.

Khi ng li bind cu hnh mi c tc dng.
